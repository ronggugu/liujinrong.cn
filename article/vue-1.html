组件的data必须是一个函数

  • 为什么组件的 data 选项必须是一个函数而 Vue 实例则没有此限制？
  • 因为组件可以被用来创建多个实例，如果 data 仍然是一个对象的话，那么所有的实例将会共享引用同一个数据对象！通过提供 data 函数，每次创建一个新实例后，我们能够调用 data 函数，从而返回初始数据的一个全新副本数据对象。
  • 你也可以在 data 选项中直接提供一个已经存在的数据对象。在这种情况下，组件实例将会“浅拷贝”其属性到对应的实例上。这意味着像 data、computed 和 methods 这样的属性将会被共享引用，而像 props 这样的属性则不会。这也意味着当一个组件的 prop 更新时，该组件内部的相应属性也将会更新。
  • 你可以通过 Vue.extend() 创建一个“子类”扩展一个组件，这样可以避免重复创建组件的 data 选项中的对象。

Vue 响应式

Vue.js 是一个构建数据驱动的 web 界面的渐进式框架。Vue.js 的核心是一个兼顾了性能与实用性的响应式数据绑定系统。当你把一个普通的 JavaScript 对象传入 Vue 实例作为它的 data 选项，Vue 将遍历此对象所有的 property，并使用 Object.defineProperty 把这些 property 全部转为 getter/setter。Object.defineProperty 是 ES5 中一个无法 shim 的特性，这也就是 Vue 不支持 IE8 以及更低版本浏览器的原因。

当你访问一个 Vue 实例上的 property 时，它会代理到对应的 data property 上，这样做的好处是我们可以在实例上直接访问 data 中的 property，同时也保证了当 data 中的 property 发生变化时，视图也会更新。这里需要注意的是，只有当实例被创建时 data 中存在的 property 才是响应式的。也就是说如果你添加一个新的 property (比如 vm.b = ‘hi’)，那么对 b 的改动将不会触发任何视图的更新。如果你知道你会在晚些时候需要一个 property，但是一开始它为空或不存在，那么你仅需要设置一些初始值。比如：

变异方法

Vue.js 的响应式系统在底层使用了 ES5 的 Object.defineProperty()。这个方法把一个普通的 JavaScript 对象转换为 getter/setter 的形式，这样就可以监听到属性的变化。但是，这个方法有一个限制，就是只能监听对象的属性，而不能监听数组的索引。因为 Vue.js 通过索引来获取数组中的成员，所以当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue，这个数组的变化不会被检测到。为了解决这个问题，Vue.js 为数组提供了一些变异方法，这些方法可以在保证响应式的情况下改变数组的值。这些方法包括：

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()
  • 这些方法都会触发视图的更新，因为它们会调用数组的变异方法，这些方法会触发视图的更新。

数据劫持

数据劫持是 Vue.js 实现响应式的基础，它的核心是通过 Object.defineProperty() 来劫持各个属性的 setter、getter，在数据变动时发布消息给订阅者，触发相应监听回调。

发布订阅模式

发布订阅模式是一种设计模式，它定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象，这个主题对象的状态发生变化时就会通知所有的观察者对象，使它们能够自动更新自己。

虚拟 DOM

虚拟 DOM 是一种数据结构，用于描述真实 DOM，有利于在内存中操作 DOM 树，最后再将变更渲染到真实 DOM 上。在 Vue.js 中，虚拟 DOM 是用 VNode 类表示的。

diff 算法

diff 算法是用于比较新旧两棵虚拟 DOM 树的算法，它的核心是通过同层级比较，找出最小的变化，最后再将变化渲染到真实 DOM 上。

模板编译

模板编译是将模板转换为渲染函数的过程，它的核心是将模板解析成 AST，再将 AST 转换为渲染函数。

依赖收集

依赖收集是在数据变化时，收集依赖该数据的所有订阅者，当数据变化时，通知所有订阅者，触发相应监听回调。

依赖更新

依赖更新是在数据变化时，通知所有订阅者，触发相应监听回调。

nextTick

nextTick 是 Vue 提供的一个异步 API，用于在 DOM 更新之后执行回调函数。这个 API 可以在一些场景下很有用，比如当我们修改数据后，想要立即获取更新后的 DOM 节点信息时。

Vue 优化

优化原则

  • 减少不必要的渲染
  • 减少不必要的侦听器
  • 减少不必要的计算属性
  • 减少不必要的 DOM 操作
  • 减少不必要的组件渲染
  • 减少不必要的依赖注入

优化技巧

  • 使用 v-if/v-else-if/v-else 替代 v-show
  • 使用 v-for 替代 v-if
  • 使用 keep-alive 替代 v-if
  • 使用 computed 替代 watch
  • 使用 v-once 替代 v-if
  • 使用 v-once 替代 v-for
