Webpack 5 详解

Webpack 是一个流行的 JavaScript 模块打包工具，它能够将多个 JavaScript 文件打包成一个或多个 JavaScript 文件，同时支持打包 CSS、图片等资源文件。Webpack 5 是最新版本的 Webpack，它带来了许多新功能和性能改进。本文将介绍如何使用 Webpack 5 来打包和管理前端项目。

常见的 Loader 和 Plugin

在 Webpack 中，Loader 和 Plugin 是两个非常重要的概念，它们的作用是帮助我们打包和优化项目。下面分别介绍一下常见的 Loader 和 Plugin。

Loader

  • babel-loader: babel-loader 是 Webpack 中使用最广泛的 Loader 之一，它的作用是将 ES6+的代码转换成浏览器能够理解的 ES5 代码。这样我们就可以在代码中使用最新的语言特性，而不用担心兼容性问题了。

  • style-loader 和 css-loader: style-loader 和 css-loader通常一起使用，它们的作用是将 CSS 文件转换成 Webpack 可识别的模块，并将其插入到 HTML 文件中。其中，css-loader负责解析 CSS 文件，并处理其依赖关系，style-loader负责将解析后的 CSS 代码插入到 HTML 文件中。

  • file-loader 和 url-loader: file-loader 和 url-loader 可以帮助我们处理图片、字体等静态资源。file-loader 会将这些资源复制到输出目录，并返回资源的路径，而 url-loader 则会将文件转换成 base64 编码的 URL，从而减少请求次数。

  • eslint-loader: eslint-loader 可以帮助我们在 Webpack 打包过程中对代码进行静态检查，以保证代码的质量和可读性。

  • ts-loader: ts-loader 可以帮助我们将 TypeScript 代码转换成 JavaScript 代码。

  • vue-loader: vue-loader 可以帮助我们将 Vue 组件转换成 JavaScript 代码。

  • postcss-loader: postcss-loader 可以帮助我们使用 PostCSS 来处理 CSS 代码。

  • sass-loader: sass-loader 可以帮助我们将 Sass 代码转换成 CSS 代码。

  • less-loader: less-loader 可以帮助我们将 Less 代码转换成 CSS 代码。

  • stylus-loader: stylus-loader 可以帮助我们将 Stylus 代码转换成 CSS 代码。

Plugin

  • html-webpack-plugin: html-webpack-plugin 可以帮助我们自动生成 HTML 文件，并将打包后的资源自动注入到 HTML 文件中。
  • mini-css-extract-plugin: mini-css-extract-plugin 可以帮助我们将 CSS 代码从 JavaScript 文件中提取出来，从而减少 JavaScript 文件的体积。
  • clean-webpack-plugin: clean-webpack-plugin 可以帮助我们在每次打包之前清空输出目录。
  • copy-webpack-plugin: copy-webpack-plugin 可以帮助我们将静态资源复制到输出目录。
  • webpack-bundle-analyzer: webpack-bundle-analyzer 可以帮助我们分析打包后的文件，以便我们更好地优化项目。
  • terser-webpack-plugin: terser-webpack-plugin 可以帮助我们压缩 JavaScript 代码。
  • optimize-css-assets-webpack-plugin: optimize-css-assets-webpack-plugin 可以帮助我们压缩 CSS 代码。
  • webpack-dev-server: webpack-dev-server 可以帮助我们启动一个开发服务器，用于开发和调试。
  • webpack-merge: webpack-merge 可以帮助我们将多个配置文件合并成一个配置文件。
  • webpackbar: webpackbar 可以帮助我们在终端中显示打包进度。
  • friendly-errors-webpack-plugin: friendly-errors-webpack-plugin 可以帮助我们在终端中显示更友好的错误信息。
  • speed-measure-webpack-plugin: speed-measure-webpack-plugin 可以帮助我们分析每个 Loader 和 Plugin 的耗时情况。
  • hard-source-webpack-plugin: hard-source-webpack-plugin 可以帮助我们缓存 Webpack 的编译结果，从而提升打包速度。
  • happypack: happypack 可以帮助我们将 Loader 转换成多进程来处理，从而提升打包速度。
  • webpack-parallel-uglify-plugin: webpack-parallel-uglify-plugin 可以帮助我们将 UglifyJS 转换成多进程来处理，从而提升打包速度。
  • webpack-dashboard: webpack-dashboard 可以帮助我们在终端中显示打包进度和错误信息。

处理 React 和 Vue

  1. 安装相应的 loader

    • 对于 React，需要安装 babel-loader，并配置相应的 Babel 插件来转译 JSX 语法和其他 ES6+ 语法。

    • 对于 Vue，需要安装 vue-loader，并配置相应的 Vue Loader 选项来转译 Vue 组件。

  2. 配置入口文件

    • 对于 React，入口文件可以是一个 JavaScript 文件，其中包含 React 组件的引用和使用。

    • 对于 Vue，入口文件可以是一个 Vue 单文件组件（.vue 文件），其中包含 Vue 组件的定义和使用。

  3. 配置输出文件

    • 输出文件通常是一个 JavaScript 文件，其中包含编译后的 React 或 Vue 组件。

    • 对于 React，输出文件可以使用 Webpack 的 output 配置项来指定。

    • 对于 Vue，输出文件通常需要在 Webpack 配置文件中使用 vue-template-compiler 插件来预编译 Vue 单文件组件，然后使用 html-webpack-plugin 插件来将编译后的模板文件注入到输出文件中。

  4. 配置相关的 Loader 和 Plugin

    • 除了 babel-loader 和 vue-loader，还需要配置其他相关的 Loader 和 Plugin，例如 css-loader、sass-loader、file-loader 等。

    • 对于 React，常用的 Plugin 包括 html-webpack-plugin、clean-webpack-plugin、copy-webpack-plugin 等。

    • 对于 Vue，常用的 Plugin 包括 vue-loader-plugin、vue-template-compiler、html-webpack-plugin

性能优化

优化构建速度

  • 优化 babel-loader：

    • 指定 babel-loader 的缓存目录：使用 babel-loader 时，可以通过指定缓存目录来避免重复编译已经编译过的代码，从而提高编译速度。在 babel-loader 的配置项中，可以设置 cacheDirectory 选项，用于指定缓存目录。

      • 使用 thread-loader 加速 babel-loader：thread-loader 是一个 Webpack 插件，用于将一些比较耗时的 Loader 放在 Worker 池中进行编译。在使用 babel-loader 时，可以将 babel-loader 放在 thread-loader 中，从而加速编译速度。
      • 只编译需要编译的代码: 在使用 babel-loader 时，通常会将所有的 ECMAScript 代码都编译为 JavaScript 代码，这可能会导致编译时间较长。为了提高编译速度，我们可以只编译需要编译的代码。
      • 使用 babel-plugin-import 优化按需加载

  • IgnorePlugin: IgnorePlugin 可以让 Webpack 忽略某些特定的模块，不进行打包和解析，从而减少打包体积和提高构建速度。

  • noParse：在 Webpack 中用来忽略对于部分非常大的第三方库（如 jQuery、lodash 等）解析的配置选项。当我们引入的第三方库已经被打包成了一个单独的文件，且不依赖其他模块时，可以在 Webpack 配置文件中使用 noParse 来提高打包速度。

  • happyPack：HappyPack 是一个能够让 webpack 在多进程环境中运行的插件，可以将 loader 的执行放在单独的 worker 线程池中进行，提高构建速度。在 webpack 构建过程中，最耗费时间的是文件解析和 loader 转换。这些工作是单线程进行的，这就导致了构建时间过长。 HappyPack 通过将这些任务分解到多个子进程中并行处理来解决这个问题。

  • ParallelUglifyPlugin: ParallelUglifyPlugin 是一个能够让 Webpack 在多进程环境下运行的插件，它会开启多个子进程，把对多个文件的压缩工作分配给多个子进程去完成，子进程处理完压缩任务后再把结果发送给主进程。这样就能充分利用多核 CPU 的优势，实现更高的构建速度。

  • 自动刷新: webpack-dev-server 会监控项目中的文件变化，当代码发生变化后，会实时重新编译，并自动刷新浏览器，让我们可以更高效地开发项目。

  • 热更新: 热更新是指在不刷新整个页面的情况下，通过使用新的模块替换旧的模块来实现页面实时更新的功能。在 Webpack 中，可以通过 HotModuleReplacementPlugin 插件来实现热更新功能。

优化产出代码

  • 使用生产环境：自动启用一些默认的优化策略，比如开启代码压缩和 scope hoisting。同时也会禁用一些只有在开发环境才有用的特性，比如 eval 方式的 source map。在不同的环境下，通过 Webpack 配置文件的不同设置，以达到最佳的构建效果。例如，在生产环境下，可以开启 tree shaking、代码分割、懒加载等优化策略，以减小构建出来的包的大小，并提高应用的加载速度。

  • 小图片使用 base64 编码： 对于一些小图片，我们可以将其转化为 base64 编码的形式，并直接内联到 HTML、CSS 或 JavaScript 中，从而减少 HTTP 请求的数量，提高页面的加载速度。Webpack 可以通过 url-loader 和 file-loader 来实现这一功能。url-loader 会将图片转换成 base64 编码，如果图片大小超过了指定的阈值，它会将图片文件输出到指定的文件夹，并在 HTML、CSS 或 JavaScript 中使用文件路径来引用。

  • bundle 加 hash: 避免缓存，在配置文件中使用 [contenthash] 表示 hash 值根据文件内容生成。而 [hash] 则是根据整个构建过程生成的 hash 值，如果只有其中一个文件发生改变，也会导致整个构建过程的 hash 值变化，不够精细。

  • 使用 CDN ：更快地加载静态资源，提高网页加载速度和用户体验。在 Webpack 中，可以通过配置 output.publicPath 来指定静态资源的 CDN 地址。

  • 提取公共改代码：一个页面可能包含多个 JavaScript 文件，如果这些文件都包含重复的代码，会导致文件体积过大，加载速度慢，影响用户体验。为了解决这个问题，Webpack 提供了代码分割和提取公共代码的功能，可以将重复代码提取到一个单独的文件中，然后在页面加载时只需加载一次该文件，从而提高页面加载速度。 提取公共代码的功能可以通过 Webpack 的 optimization 配置来实现，其中 splitChunks 用于配置代码分割，runtimeChunk 用于配置运行时代码分割。

polyfill

Polyfill 是指在老旧浏览器上实现新的 API 或者 ECMAScript 特性的代码库。在开发现代 web 应用程序时，我们会使用许多新的 API 和语言特性。但是，这些功能并不总是在所有浏览器中可用，这就是需要 polyfill 的地方。

例如，ES6 新增了许多强大的特性，例如箭头函数、解构、let/const、Promise、模板字符串等。但是，这些新的特性在一些老旧的浏览器中并不被支持。为了让这些浏览器也能正常运行我们的代码，我们需要使用 Polyfill。

Polyfill 的作用是模拟新 API 或特性，使得旧版本的浏览器也能够支持。它可以在代码中注入代码片段、重写方法、对象、属性，以达到支持新特性的目的。

可以在 webpack 中使用 polyfill 实现兼容性处理，通过 babel-polyfill 或者 core-js 等库提供的 polyfill 来兼容旧版浏览器中不支持的新特性。

runtime

Webpack 5 会在每个 chunk 中插入 runtime 代码，用于管理模块之间的交互。这样可以避免不同模块之间的重复代码，从而减少 bundle 的大小。

在使用了 SplitChunks 插件之后，可以看到生成的文件中会有一个名为 runtime.[hash].js 的文件，该文件包含了管理模块之间交互的代码。
