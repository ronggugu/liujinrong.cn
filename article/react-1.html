⏰生命周期

类组件的生命周期

React组件的生命周期可以分为三个阶段：挂载（Mounting）、更新（Updating）和卸载（Unmounting）。每个阶段都有相应的生命周期函数，可以在不同阶段执行特定的操作。

  • 挂载阶段

    • 在组件被创建并插入到DOM中时，会经历挂载阶段。以下是挂载阶段的生命周期函数：
    • constructor：组件的构造函数，在组件被创建时调用，用于初始化状态和绑定方法。
    • render：必须的生命周期函数，在该函数中返回组件的JSX结构。
    • componentDidMount：组件被插入到DOM后调用，通常用于进行数据获取或订阅事件。

  • 更新阶段
    - 当组件的状态或属性发生变化时，会触发更新阶段。以下是更新阶段的生命周期函数：
    - static getDerivedStateFromProps：该静态函数在组件接收到新的属性时调用，用于根据新的属性值更新状态。
    - shouldComponentUpdate：决定组件是否需要进行重新渲染，可以在此函数中进行性能优化。
    - render：重新渲染组件的JSX结构。
    - getSnapshotBeforeUpdate：在组件更新之前获取DOM快照，常用于获取滚动位置等信息。
    - componentDidUpdate：组件更新完成后调用，通常用于处理更新后的副作用操作。

  • 卸载阶段

    • 当组件从DOM中移除时，会经历卸载阶段。以下是卸载阶段的生命周期函数：
    • componentWillUnmount：组件即将从DOM中移除时调用，通常用于清理资源和取消订阅事件。

函数组件的生命周期

高级使用

1. 受控组件和非受控组件

受控组件

受控组件是指表单数据由 React 组件处理的组件。表单元素的值由 React 的状态控制，当状态发生变化时，表单元素的值也发生变化，从而实现了双向绑定。

非受控组件

非受控组件是指表单数据由 DOM 元素处理的组件。表单元素的值由 DOM 元素本身处理，当 DOM 元素的值发生变化时，表单元素的值也发生变化，从而实现了双向绑定。

2. 高阶组件

高阶组件是指接收组件作为参数并返回新组件的函数。高阶组件可以用于代码复用、逻辑抽象、状态管理等场景。

3. refs

refs 是 React 提供的获取 DOM 元素或组件实例的方法。在函数组件中，可以使用 useRef Hook 来获取 DOM 元素或组件实例。

4. context

context 是 React 提供的一种跨组件传递数据的方法。在函数组件中，可以使用 useContext Hook 来获取 context 中的数据。

5. Portals

Portals 是 React 提供的一种将子组件渲染到父组件 DOM 以外的方法。在函数组件中，可以使用 ReactDOM.createPortal 方法来实现 Portals。

6. 异步组件

异步组件是指在需要时才加载的组件。在函数组件中，可以使用 React.lazy 方法来实现异步组件。

7. 性能优化

  • React 提供了一些性能优化的方法，如 shouldComponentUpdate、React.memo、useMemo、useCallback 等。
  • 在函数组件中，可以使用 React.memo 方法来实现 shouldComponentUpdate 的功能。
  • 在函数组件中，可以使用 useMemo 和 useCallback 来缓存计算结果。
  • 在函数组件中，可以使用 useReducer 来代替 useState，从而避免 useState 的性能问题。
  • 在函数组件中，可以使用 useLayoutEffect 来代替 useEffect，从而避免 useEffect 的性能问题。

8. render props

render props 是指将组件作为参数的函数。在函数组件中，可以使用 children 属性来实现 render props。

周边工具

1. React Router

React Router 是 React 提供的一种路由管理的方法。在函数组件中，可以使用 react-router-dom 提供的各种组件来实现路由管理。

2. Redux

Redux 是 React 提供的一种状态管理的方法。在函数组件中，可以使用 react-redux 提供的各种组件来实现状态管理。

原理

1. JSX

JSX 是 React 提供的一种声明式的语法。在函数组件中，可以使用 JSX 语法来声明组件。JSX 本质上是 React.createElement

2. vdom 和 diff

vdom 是 React 提供的一种虚拟 DOM 的实现。在函数组件中，可以使用 vdom 来实现组件的渲染。
diff 是 React 提供的一种虚拟 DOM 的比较算法。在函数组件中，可以使用 diff 来实现组件的更新。
vdom 本质上是一个普通的 JavaScript 对象，diff 本质上是一个普通的 JavaScript 函数。

3. Hooks

Hooks 是 React 提供的一种状态管理的方法。在函数组件中，可以使用 Hooks 来实现状态管理。
Hooks 本质上是一个普通的 JavaScript 函数。

4. 合成事件

合成事件是 React 提供的一种事件处理的方法。在函数组件中，可以使用合成事件来处理 DOM 事件。event 本质上是一个普通的 JavaScript 对象。react 会将原生事件封装成合成事件，从而实现跨平台的兼容性。 获取原生事件对象，可以使用 event.nativeEvent。

5. 组件渲染过程

组件渲染过程分为两个阶段：mounting 和 updating。

分为三个步骤：render、commit、effect。

  • render 阶段是指调用函数组件，生成 vdom 的过程。

  • commit 阶段是指将 vdom 转换成 dom 的过程。

  • effect 阶段是指执行副作用的过程。

6. 前端路由

前端路由的原理是基于浏览器提供的history API（HTML5 History API）和pushState方法。这些API允许我们修改浏览器的URL，并将修改后的URL添加到浏览器的访问历史记录中。

React Router通过使用history API来实现前端路由。它提供了BrowserRouter组件和Route组件，用于将URL与组件进行关联，并在URL发生变化时渲染相应的组件。

当用户点击应用中的链接或通过编程方式改变URL时，React Router会通过history API将新的URL添加到浏览器的历史记录中。然后，React Router会根据配置的路由规则，匹配当前URL与相应的组件，并渲染匹配的组件。
