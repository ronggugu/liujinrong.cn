前端面试总结

HTML/CSS

1. 什么是盒模型？有哪几种？

盒模型是指 HTML 元素在页面中表现为一个矩形的模型，它由四部分组成：content、padding、border 和 margin。其中，content 是元素的实际内容，padding 是元素的内边距，border 是元素的边框，margin 是元素的外边距。这四部分属性可以通过 CSS 的 width、height、padding、border 和 margin 属性进行控制。

2. 什么是 BFC？有哪些应用？

BFC 是块级格式化上下文，它是一个独立的渲染区域，只有 Block-level Box 参与，它规定了内部的 Block-level Box 如何布局，并且与这个区域外部毫不相干。它是一个独立的渲染区域，只有 Block-level Box 参与，它规定了内部的 Block-level Box 如何布局，并且与这个区域外部毫不相干。

BFC 的布局规则：

  • 内部的 Box 会在垂直方向，一个接一个地放置。
  • Box 垂直方向的距离由 margin 决定。属于同一个 BFC 的两个相邻 Box 的 margin 会发生重叠。
  • 每个元素的 margin box 的左边，与包含块 border box 的左边相接触（对于从左往右的格式化，否则相反），即使存在浮动也是如此。
  • BFC 的区域不会与 float box 重叠。
  • BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。
  • 计算 BFC 的高度时，浮动元素也参与计算。

BFC 的应用：

  • 解决 margin 重叠问题
  • 解决浮动元素影响布局问题
  • 解决多列布局问题

3. 什么是 CSS 选择器？有哪些？

CSS 选择器是用来选取 HTML 元素的，它可以根据 HTML 元素的 id、class、标签名、属性等来选取元素，从而对元素进行样式设置。

CSS 选择器有以下几种：

  • 元素选择器：通过元素名来选取元素，如 h1、p、div 等。
  • id 选择器：通过元素的 id 属性来选取元素，id 属性在页面中是唯一的，因此 id 选择器在页面中只能选取一个元素。
  • class 选择器：通过元素的 class 属性来选取元素，class 属性可以为元素设置多个 class，因此 class 选择器可以选取多个元素。
  • 属性选择器：通过元素的属性来选取元素，如 a[href]、input[type=”text”] 等。
  • 伪类选择器：通过元素的特殊状态来选取元素，如 a:hover、input:focus 等。
  • 伪元素选择器：通过元素的特殊部分来选取元素，如 p::first-line、p::first-letter 等。
  • 组合选择器：通过组合不同的选择器来选取元素，如 div p、div > p、div + p 等。
  • 通配符选择器：通过 * 选择所有元素。

4. 什么是 CSS 预处理器？有哪些？

CSS 预处理器是一种编程语言，它可以扩展 CSS 的语法，增加了变量、函数、继承等特性，使 CSS 更易维护和扩展。CSS 预处理器有以下几种：

  • Less：Less 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。
  • Sass：Sass 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。
  • Stylus：Stylus 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。

5. 什么是 CSS Sprite？有哪些优缺点？

CSS Sprite 是一种将多个小图标合并到一张图片中的技术，然后通过 CSS 的 background-position 属性来显示合并后图片中的某个区域，从而达到显示多个小图标的目的。CSS Sprite 可以减少 HTTP 请求，提高页面加载速度，但是维护成本较高。

6. 什么是响应式布局？有哪些优缺点？

响应式布局是一种 Web 页面布局方式，它可以根据不同的屏幕尺寸，自动调整页面的布局，从而达到适配不同屏幕尺寸的目的。响应式布局的优点是可以适配不同屏幕尺寸，缺点是开发成本较高。

7. 什么是 CSS 布局？有哪些？

CSS 布局是指通过 CSS 来控制页面元素的布局，包括元素的位置、大小、外边距、内边距、边框等。CSS 布局有以下几种：

  • 浮动布局：通过浮动元素来实现元素的布局。
  • 定位布局：通过绝对定位和相对定位来实现元素的布局。
  • Flex 布局：通过 Flex 布局来实现元素的布局。
  • Grid 布局：通过 Grid 布局来实现元素的布局。

8. 什么是 CSS 动画？有哪些？

CSS 动画是指通过 CSS 来控制页面元素的动画效果，包括元素的位置、大小、外边距、内边距、边框、透明度、颜色等。CSS 动画有以下几种：

  • transition：通过 transition 属性来实现元素的动画效果。
  • animation：通过 animation 属性来实现元素的动画效果。
  • transform：通过 transform 属性来实现元素的动画效果。

9. 如何实现一个水平居中或垂直居中的元素？

水平居中可以使用 text-align:center；或者 margin-left:auto;margin-right:auto；实现。垂直居中可以使用绝对定位和负 margin 实现，或者使用 display:flex 和 align-items:center 实现。

10. 什么是浮动？如何清除浮动？

浮动是指使元素脱离文档流，向左或向右移动，直到它的外边缘碰到了包含它的容器或另一个浮动元素为止。清除浮动可以使用清除浮动的方法，包括在浮动元素下方添加一个空元素，并设置 clear:both；或者使用 overflow:hidden；或者使用 :after 伪元素清除浮动。

11. 重绘和回流是什么？如何避免？

重绘和回流是浏览器渲染页面的两个过程。重绘和回流的关系，回流一定会引起重绘，但是重绘不一定会引起回流。比如改变元素的位置、大小、颜色、内容、字体等，都会引起重绘和回流。改变元素的背景色、边框颜色、字体颜色等，都只会引起重绘。
重绘是指当元素的外观发生改变，但是它的布局位置没有发生改变时，浏览器需要重新绘制元素的外观，这个过程称为重绘。回流是指当元素的布局位置发生改变时，浏览器需要重新计算元素的位置，然后再绘制元素，这个过程称为回流。可以通过以下几种方式避免重绘和回流：

  • 将 DOM 离线后修改，修改完成后再把 DOM 添加到文档中。
  • 使用 documentFragment 对象，先将 DOM 添加到 documentFragment 对象中，然后再把 documentFragment 对象添加到文档中。
  • 使用 display:none；或者 visibility:hidden；来隐藏元素，使用 display:block；或者 visibility:visible；来显示元素。
  • 使用 position:fixed；或者 position:absolute；来定位元素，使用 position:static；来取消定位。
  • 使用 transform:translateZ(0)；或者 transform:translate3d(0,0,0)；来开启 GPU 加速。
  • 使用 will-change:transform；来开启 GPU 加速。
  • 使用 opacity:0；来隐藏元素，使用 opacity:1；来显示元素。
  • 使用 z-index:-1；来隐藏元素，使用 z-index:auto；来显示元素。
  • 使用 CSS 动画来代替 JavaScript 动画。
  • 使用 CSS3 的新属性来代替 JavaScript 动画。

12.flex 布局有哪些属性？

  • flex-direction：定义主轴的方向。
  • flex-wrap：定义是否换行。
  • flex-flow：flex-direction 和 flex-wrap 的简写。
  • justify-content：定义项目在主轴上的对齐方式。
  • align-items：定义项目在交叉轴上的对齐方式。
  • align-content：定义多根轴线的对齐方式。
  • order：定义项目的排列顺序。数值越小，排列越靠前，默认为 0。
  • flex-grow：定义项目的放大比例，默认为 0，即如果存在剩余空间，也不放大。
  • flex-shrink：定义了项目的缩小比例，默认为 1，即如果空间不足，该项目将缩小。
  • flex-basis：定义了在分配多余空间之前，项目占据的主轴空间（main size）。
  • flex：flex-grow、flex-shrink 和 flex-basis 的简写，默认值为 0 1 auto。
  • align-self：允许单个项目有与其他项目不一样的对齐方式，可覆盖 align-items 属性。

二、JavaScript 常用的面试题

1. 什么是 JavaScript？

JavaScript 是一种脚本语言，是一种基于原型、函数先行的语言，是一门多范式的语言，它支持面向对象编程，命令式编程，以及函数式编程。它提供语法来操控文本、数组、日期以及正则表达式等，不支持 I/O，比如网络、存储和图形等，但这些都可以由它的宿主环境提供支持。它已经由 ECMA（欧洲计算机制造商协会）通过 ECMA-262 标准化。

2. 什么是 ECMAScript？

ECMAScript 是一种由 ECMA（欧洲计算机制造商协会）制定的脚本语言标准，它是 JavaScript 的规范，也是 JavaScript 的基础。

3. 什么是 DOM？

DOM 是指文档对象模型（Document Object Model），它是 HTML 和 XML 文档的编程接口。DOM 将整个页面抽象为一个由多层节点构成的集合。每个节点都是一个对象，它可以拥有属性和方法，可以通过 JavaScript 来操作 DOM。

4. 什么是 BOM？

BOM 是指浏览器对象模型（Browser Object Model），它提供了与浏览器交互的方法和接口。BOM 由 window、navigator、screen、location、history、frames 等对象组成。

5. 什么是作用域？

作用域是指程序源代码中定义变量的区域。JavaScript 中有两种作用域：全局作用域和局部作用域。

6. 什么是闭包？

闭包是指有权访问另一个函数作用域中的变量的函数。创建闭包的常见方式，就是在一个函数内部创建另一个函数。

7. 什么是原型链？

原型链是指对象的原型对象又是另一个对象的原型对象，这样一层层的链接起来的对象组成的链条，这个链条就是原型链。

8. 什么是事件冒泡？

事件冒泡是指事件开始时由最具体的元素接收，然后逐级向上传播到较为不具体的节点（文档）。事件冒泡的本质是事件的传播机制。

9. 什么是事件委托？

事件委托是指将事件绑定到父元素上，利用事件冒泡的原理，触发事件时，事件会从子元素向父元素传播，从而触发父元素的事件处理函数。

10. 什么是 JSON？

JSON 是指 JavaScript 对象表示法（JavaScript Object Notation），它是一种轻量级的数据交换格式。JSON 采用完全独立于语言的文本格式，但是也使用了类似于 C 语言家族的习惯（包括 C、C++、C#、Java、JavaScript、Perl、Python 等）。这些特性使 JSON 成为理想的数据交换语言。

11. 什么是 Ajax？

Ajax 是指异步 JavaScript 和 XML（Asynchronous JavaScript and XML），它是一种创建交互式网页应用的网页开发技术。Ajax 不是新的编程语言，而是一种使用现有标准的新方法。

12. 什么是跨域？

跨域是指浏览器的同源策略限制了从一个源加载的文档或脚本如何与另一个源的资源进行交互。跨域资源共享（CORS）是一种机制，它使用额外的 HTTP 头来告诉浏览器 让运行在一个 origin（域名+协议+端口）上的 Web 应用被准许访问来自不同源服务器上的指定的资源。

13. 什么是同源策略？

同源策略是指一个源的文档或脚本如何来自另一个源的资源进行交互。它是一个用于隔离潜在恶意文件的关键的安全机制。同源策略是浏览器最核心也是最基本的安全功能，如果同源策略失效，浏览器很容易受到 XSS 和 CSRF 攻击。所以，同源策略是浏览器最核心也是最基本的安全功能。

14. 什么是 XSS 攻击？

XSS 攻击是指跨站脚本攻击（Cross Site Scripting），它是一种代码注入攻击。攻击者往 Web 页面里插入恶意的 HTML 代码，当用户浏览该页之时，嵌入其中 Web 里面的 JavaScript 代码会被执行，从而达到恶意攻击用户的目的。

15. 什么是 CSRF 攻击？

CSRF 攻击是指跨站请求伪造（Cross Site Request Forgery），它是一种挟制用户在当前已登录的 Web 应用程序上执行非本意的操作的攻击方法。CSRF 攻击通常利用网站对用户的信任来产生作用。

16. 什么是 Promise？

Promise 是一种异步编程的解决方案，它是异步编程的一种新的解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。Promise 对象代表一个异步操作，有三种状态：pending（进行中）、fulfilled（已成功）和 rejected（已失败）。

17. 什么是 async/await？

async/await 是 ES2017 引入的新的语法，它是异步编程的一种新的解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。async/await 是基于 Promise 实现的，它不能用于普通的回调函数。

18. 什么是模块化？

模块化是指将一个复杂的系统分解成一个个相互依赖的模块，以便于编码和维护。模块化可以提高代码的可维护性和复用性，也有利于代码的分工协作。

19. 什么是 MVC？

MVC 是指模型（Model）、视图（View）和控制器（Controller），它是一种软件设计模式。MVC 将一个应用程序分为三个基本部分：模型（Model）、视图（View）和控制器（Controller）。

20. 什么是 MVVM？

MVVM 是指模型（Model）、视图（View）和视图模型（ViewModel），它是一种软件设计模式。MVVM 将一个应用程序分为三个基本部分：模型（Model）、视图（View）和视图模型（ViewModel）。

21. 什么是单例模式？

单例模式是指保证一个类仅有一个实例，并提供一个访问它的全局访问点。

22. 什么是发布订阅模式？

发布订阅模式是指定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到通知。

23. 什么是观察者模式？

观察者模式是指定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到通知。

24. 什么是闭包？

闭包是指有权访问另一个函数作用域中的变量的函数。创建闭包的常见方式，就是在一个函数内部创建另一个函数。

25. 什么是原型链？

原型链是指每个对象都有一个原型对象（prototype），对象的原型对象也可能有自己的原型对象，这样一直延续下去，直到某个原型对象的原型对象为 null。根据定义，null 没有原型，并作为这个原型链中的最后一个环节。

26. 什么是作用域链？

作用域链是指作用域链是指在代码中定义变量的位置决定了它的可见范围，这个可见范围就是它的作用域。JavaScript 采用词法作用域，也就是静态作用域。词法作用域意味着作用域是由书写代码时函数声明的位置来决定的。

27. 什么是事件循环？

事件循环是指 JavaScript 是单线程的，但它通过事件循环来实现异步编程。事件循环是指 JavaScript 引擎有一个主线程，主线程之外还有一个任务队列，当主线程中的任务执行完毕后，会从任务队列中读取任务放到主线程中执行。

28. 什么是事件委托？

事件委托是指利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。事件委托的好处是可以减少内存占用，提高性能。

29. 什么是防抖和节流？

防抖和节流是指防抖（debounce）和节流（throttle）是两种常用的函数节流技术，它们都是为了解决高频率事件触发时带来的性能问题。

30. 什么是深拷贝和浅拷贝？

深拷贝和浅拷贝是指深拷贝（deep copy）和浅拷贝（shallow copy）是两种常用的对象拷贝技术，它们都是为了解决对象拷贝时带来的性能问题。

31. 什么是 JSONP？

JSONP 是指 JSON with Padding，它是一种跨域通信的解决方案。JSONP 的原理是利用 <script> 标签没有跨域限制的漏洞，网页可以得到从其他来源动态产生的 JSON 数据。这种方法只支持 GET 请求，不支持 POST 等其它类型的 HTTP 请求。

32. 什么是跨域？

跨域是指浏览器的同源策略限制了从一个源加载的文档或脚本如何与另一个源的资源进行交互。跨域资源共享（CORS）是一种机制，它使用额外的 HTTP 头来告诉浏览器 该访问哪些源。

33. 什么是同源策略？

同源策略是指浏览器的同源策略限制了从一个源加载的文档或脚本如何与另一个源的资源进行交互。同源策略是一种约定，它是浏览器最核心也最基本的安全功能，如果缺少了同源策略，浏览器的正常功能可能都会受到影响。它用于隔离潜在恶意文件，使其只能访问与自己相同源的资源。

34. 什么是 XSS？

XSS 是指跨站脚本攻击（Cross Site Scripting），它是一种代码注入攻击。XSS 攻击通常包含三个要素：恶意脚本、受害者的浏览器和受害者的网站。XSS 攻击的目的是为了盗取用户的 Cookie、Session、账号信息等敏感信息，或者利用受害者的权限执行恶意操作。

35. 什么是 CSRF？

CSRF 是指跨站请求伪造（Cross Site Request Forgery），它是一种代码注入攻击。CSRF 攻击通常包含三个要素：恶意网站、受害者的浏览器和受害者的网站。CSRF 攻击的目的是为了盗取用户的 Cookie、Session、账号信息等敏感信息，或者利用受害者的权限执行恶意操作。

36. 什么是 DOM？

DOM 是指文档对象模型（Document Object Model），它是 HTML 和 XML 文档的编程接口。DOM 将整个页面抽象为一个由多层节点构成的结构，通过 DOM，可以对页面中的各个元素进行增删改查。

37. 什么是 BOM？

BOM 是指浏览器对象模型（Browser Object Model），它是浏览器提供的一组对象，用于访问浏览器的功能。BOM 包含了浏览器窗口、框架、历史、位置、导航和屏幕等对象。

38. 什么是 HTTP？

HTTP 是指超文本传输协议（HyperText Transfer Protocol），它是一种无状态的、应用层的协议。HTTP 协议是基于 TCP/IP 协议的应用层协议，它规定了浏览器和 Web 服务器之间互相通信的规则。

39. 什么是 HTTPS？

HTTPS 是指安全超文本传输协议（Secure HyperText Transfer Protocol），它是一种安全的、无状态的、应用层的协议。HTTPS 协议是基于 HTTP 协议的，它在 HTTP 协议的基础上加入了 SSL/TLS 协议，从而实现了安全传输。

40. 什么是 TCP？

TCP 是指传输控制协议（Transmission Control Protocol），它是一种面向连接的、可靠的、基于字节流的传输层协议。TCP 协议是一种面向连接的协议，它提供了可靠的字节流服务，通过 TCP 连接传送的数据，无差错、不丢失、不重复、并且按序到达。

41. 什么是 UDP？

UDP 是指用户数据报协议（User Datagram Protocol），它是一种无连接的、不可靠的、基于数据报的传输层协议。UDP 协议是一种无连接的协议，它不需要建立连接，只需要知道对方的 IP 地址和端口号，就可以直接发送数据包。UDP 协议不保证数据的可靠性，因此主要用于一次只传送少量数据的应用场景，如视频会议、实时游戏等。

42. 什么是 TCP 三次握手？

TCP 三次握手是指 TCP 协议建立连接时的三次交互过程。TCP 三次握手的目的是为了防止已失效的连接请求报文段突然又传送到了服务端，因而产生错误。

43. 什么是 TCP 四次挥手？

TCP 四次挥手是指 TCP 协议断开连接时的四次交互过程。TCP 四次挥手的目的是为了双方都能及时释放资源，防止资源浪费。

44. 什么是 HTTP 状态码？

HTTP 状态码是指 HTTP 协议的响应状态码，它是一个三位数的整数，用来表示客户端 HTTP 请求的返回结果。HTTP 状态码分为五类，分别是 1xx、2xx、3xx、4xx 和 5xx。

45. 什么是 HTTP 缓存？

HTTP 缓存是指浏览器缓存机制，它是一种优化网站性能的手段。HTTP 缓存可以减少网络请求，从而提高网站的访问速度。

46. 什么是 HTTP 长连接？

HTTP 长连接是指 HTTP 协议的持久连接，它是一种优化网站性能的手段。HTTP 长连接可以减少 TCP 连接的建立和断开所带来的开销，从而提高网站的访问速度。

47. 什么是 HTTP2？

HTTP2 是指 HTTP/2，它是 HTTP 协议的下一代标准。HTTP2 是一个二进制协议，它在传输层使用了新的二进制格式，而不是 HTTP1.x 中的文本格式。HTTP2 采用多路复用技术，可以在一个连接上同时处理多个请求，从而提高网站的访问速度。

48. 什么是 HTTPS？

HTTPS 是指安全超文本传输协议（Secure HyperText Transfer Protocol），它是一种安全的、无状态的、应用层的协议。HTTPS 协议是基于 HTTP 协议的，它在 HTTP 协议的基础上加入了 SSL/TLS 协议，从而实现了安全传输。

49. 什么是 SSL？

SSL 是指安全套接层（Secure Sockets Layer），它是一种安全的、应用层的协议。SSL 协议是基于 HTTP 协议的，它在 HTTP 协议的基础上加入了 SSL 协议，从而实现了安全传输。

50. 什么是 TLS？

TLS 是指传输层安全（Transport Layer Security），它是一种安全的、应用层的协议。TLS 协议是基于 HTTP 协议的，它在 HTTP 协议的基础上加入了 TLS 协议，从而实现了安全传输。

51. 什么是 SSL/TLS 握手？

SSL/TLS 握手是指 SSL/TLS 协议建立连接时的交互过程。SSL/TLS 握手的目的是为了双方都能确认对方的身份，防止遭遇中间人攻击。

52. 什么是 CORS？

CORS 是指跨域资源共享（Cross-Origin Resource Sharing），它是一种浏览器的安全策略。CORS 限制了从一个域名下的网页去请求另一个域名下的资源，从而防止了恶意网站窃取数据。
